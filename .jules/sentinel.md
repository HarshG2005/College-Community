## 2024-05-18 - [ReDoS and NoSQL Injection in Placements Route]
**Vulnerability:** The `/api/placements` route was vulnerable to Regular Expression Denial of Service (ReDoS) due to passing unescaped user input (`req.query.company`) directly into a `new RegExp` constructor. Additionally, the route was vulnerable to NoSQL injection and application crashes because `req.query` parameters (`type`, `company`, `search`) were not explicitly cast to strings, allowing object payloads (e.g., `?type[$ne]=all`) to bypass logic or cause `.toLowerCase()` to fail on objects.
**Learning:** In Node.js/Express, `req.query` parameters can be parsed as objects or arrays if the query string is formatted a certain way (e.g., `?param[key]=value`). Passing these objects to functions expecting strings (like string methods or `RegExp`) can lead to vulnerabilities or crashes. Always assume query parameters might not be strings.
**Prevention:** Always explicitly cast `req.query` and `req.params` parameters to strings (e.g., `String(param)`) before using them. When using user input to construct a regular expression via `new RegExp`, always escape special regex characters (e.g., `input.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`) to prevent ReDoS attacks.
