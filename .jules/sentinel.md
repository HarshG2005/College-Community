## 2024-05-18 - Prevent ReDoS and Type DoS in search queries
**Vulnerability:** `req.query` inputs like `company`, `type`, and `search` were used unvalidated in database queries and native JavaScript functions (`new RegExp`, `.toLowerCase()`). This exposes the application to Type DoS (sending object inputs that cause `.toLowerCase()` to crash) and Regular Expression Denial of Service (ReDoS) because user-provided strings could include unescaped regex characters, hanging the Node.js event loop during Mongoose `findOne`/`find`.
**Learning:** `req.query` parameters are implicitly cast to objects by Express if passed as `?param[$ne]=val`. String functions fail on such objects, and raw user strings injected into `new RegExp` create a path for DoS. The vulnerability pattern of raw injection into Mongoose queries creates critical risks across the entire Express layer since no global security middleware intercepts this.
**Prevention:** Always explicitly cast `req.query` inputs to strings (e.g., `String(param)`) before using them. Escape special regex characters in any user string intended for `new RegExp` using `.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`.
