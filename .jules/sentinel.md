## 2024-05-24 - [ReDoS and NoSQL Injection via Unsanitized Express Query Parameters]
**Vulnerability:** A Regex Denial of Service (ReDoS) vulnerability in `server/routes/placements.js` where user input `req.query.company` was passed directly into `new RegExp(company, 'i')`. Additionally, passing objects like `?type[$ne]=` caused DoS when `.toLowerCase()` was called, and NoSQL injection risk by injecting operators into `.find()`.
**Learning:** In Express, query parameters and route parameters can be arrays or objects instead of strings. Passing objects to string methods (like `.toLowerCase()`) throws an error causing DoS, and passing user input directly into `new RegExp()` allows malicious regex patterns to be executed, which can halt the thread. Mongoose string-type fields also don't automatically protect against passing query objects like `{ $ne: null }` if not explicitly cast to a string first.
**Prevention:** Always explicitly cast `req.query` and `req.params` variables to strings (e.g., `String(param)`) before using them. When using user input to create a Regular Expression, it must be escaped using `.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` to neutralize special regex characters.
