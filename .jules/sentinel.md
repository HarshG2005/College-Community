## 2024-05-24 - ReDoS and NoSQL Injection in Express Routes
**Vulnerability:** The `GET /api/placements` route passed raw `req.query` parameters directly into database queries (e.g., `query.$text = { $search: search }` and `new RegExp(company, 'i')`) and string methods (e.g., `type.toLowerCase()`). This allows NoSQL injection via object payloads (`?search[$ne]=test`) and ReDoS if the `company` string contains unescaped regex characters, as well as DoS if string methods are called on an object payload.
**Learning:** `req.query` and `req.params` in Express can be parsed as objects instead of strings if the client sends array or object syntax (e.g., `?param[]=val` or `?param[key]=val`). Additionally, user input passed to `new RegExp` must always be escaped.
**Prevention:** Always explicitly cast `req.query` and `req.params` parameters to strings using `String(param)` before using them in database queries or string methods. Always escape user input using `.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` before passing it to `new RegExp`.
