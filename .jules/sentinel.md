## 2024-05-24 - [ReDoS and NoSQL Injection from Uncast Query Parameters]
**Vulnerability:** Express routes (e.g., in `server/routes/placements.js`) passed uncast `req.query` and `req.params` values directly into Mongoose queries and `new RegExp()`. If an attacker sent an object payload (like `?company[$ne]=x`) or a crafted regex string, it could cause Regex Denial of Service (ReDoS) or bypass query logic (NoSQL injection).
**Learning:** By default, Express parses query parameters as objects if they contain brackets. String methods like `.toLowerCase()` or `.replace()` will throw TypeErrors on these objects, leading to application crashes. Furthermore, unescaped user input passed to `new RegExp` is a classic ReDoS vector.
**Prevention:** Always explicitly cast user input from `req.query` and `req.params` to `String()` before applying string methods or using them in database queries. Always escape strings before using them in a regular expression (e.g., `input.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`).
