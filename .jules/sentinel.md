## 2024-05-24 - [ReDoS and NoSQL Injection in Express Routes]
**Vulnerability:** A placement filtering route (`server/routes/placements.js`) took string input directly from `req.query.company` and fed it into a `new RegExp(company, 'i')` constructor. This introduced a Regex Denial of Service (ReDoS) vulnerability if a malicious user provided a complex regex string. Additionally, since input was coming from `req.query`, not casting to a string meant an attacker could supply an array or an object (e.g. `?type[$ne]=xxx`), leading to an application crash (DoS) when `.toLowerCase()` is called on it.
**Learning:** In standard Express routes, input parameters in `req.query` or `req.params` could be objects or arrays unless strictly typed or validated. Using regex with unescaped user input opens a massive ReDoS attack surface.
**Prevention:** Always explicitly cast user input to a String (e.g. `String(req.query.company)`) and apply an escaping function replacing special characters (e.g. `replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`) before passing it into `new RegExp()`. Always explicitly cast inputs before invoking string methods like `.toLowerCase()`.
