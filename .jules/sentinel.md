## 2024-05-24 - [ReDoS and NoSQL DoS via Unescaped Regex and Object Query Parameters]
**Vulnerability:** User input from `req.query` was being directly used to construct a `new RegExp` for Mongoose queries without sanitization, leading to a potential Regular Expression Denial of Service (ReDoS). Additionally, `req.query` parameters were not cast to strings, allowing attackers to pass objects (e.g., `?param[$ne]=val` or arrays) causing potential NoSQL injection or application crashes (DoS) when `.toLowerCase()` or similar string methods were called.
**Learning:** Always treat Express `req.query` parameters as potentially dangerous inputs. Express can parse objects and arrays from the query string. Unescaped input in `new RegExp` allows attackers to craft complex regex patterns that freeze the application's event loop.
**Prevention:** Explicitly cast URL parameters to strings (e.g., `String(req.query.param)`) when expecting text input. Always escape user input using a regex escaper (e.g., `input.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`) before using it in a `new RegExp` for database queries.
