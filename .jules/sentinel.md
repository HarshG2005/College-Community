## 2024-05-28 - [ReDoS and DoS via Express Query Parameters]
**Vulnerability:** The application was passing `req.query.company` directly into `new RegExp(company, 'i')` which allows for Regular Expression Denial of Service (ReDoS) if malicious regex payload is provided by user. Also, `req.query.type.toLowerCase()` can cause application crash (DoS) if an array or object is passed via the query string.
**Learning:** In Express, `req.query` and `req.body` parameters can be parsed as arrays or objects instead of strings. Direct invocation of string-specific methods (`toLowerCase()`) on these values can lead to unexpected crashes. Passing unsanitized input directly into `new RegExp` allows malicious actors to craft expressions that hang the Node.js event loop due to catastrophic backtracking.
**Prevention:** Always cast untrusted user input from `req.query`, `req.params`, or `req.body` to a string using `String()` before calling string methods. Always sanitize input by escaping regex special characters `(String(input).replace(/[.*+?^${}()|[\]\\]/g, '\\$&'))` before passing it to `new RegExp`.
